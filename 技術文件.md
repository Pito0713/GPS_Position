## 最重要的底層事實

```
iOS DVT simulate-location set：
  ✅ process 存活  →  虛擬定位持續
  ✅ process 死亡  →  立刻跳回真實位置
  ✅ 兩個 process 並存  →  iOS 採用最新的，不閃爍
  ✅ Pool 最多保留 2 個存活 process（防止 fd 耗盡）
  ✅ 路線中絕不 kill 任何 process
```

```
xcrun simctl location set（Simulator）：
  ✅ 立刻生效，無需 sleep
  ✅ 無 process pool 問題
  ✅ 最小間隔 0.5s（實機需要 2.0s）
```

## 後端 `gps_spoofer.py`

### 全域狀態

```python
_proc_pool    # list[Popen]  存活的 simulate-location set processes
_pool_lock    # Lock         保護 _proc_pool

route_thread  # 路線執行緒（daemon）
route_stop    # Event  停止
route_pause   # Event  暫停
route_speed   # float  即時速度（每步讀取）
route_state   # dict   前端輪詢用
```

### `hold_coord(lat, lng)` — 核心

```python
proc = Popen(["simulate-location", "set", ..., lat, lng])

# 存活確認（0.3s）
time.sleep(0.3)
if proc.poll() is not None: return False  # 啟動失敗

# 等 iOS 確認（0.3 → 1.5s 共等 1.2s）
time.sleep(1.2)

# Pool 操作
with _pool_lock:
    _proc_pool.append(proc)
    alive   = [p for p in _proc_pool if p.poll() is None]
    to_kill = alive[:-2]   # 保留最新 2 個
    _proc_pool = alive[-2:]

# 非阻塞 kill（背景執行緒）
for p in to_kill: _kill(p)
```

### `_kill(proc)` — 非阻塞

```python
def _kill(proc):
    def _bg():
        proc.terminate()
        proc.wait(timeout=3)  # 超時再 kill
    threading.Thread(target=_bg, daemon=True).start()
```

路線執行緒不會被 kill 阻塞。

### `_do_step(lat, lng, ivl)` — 每步

```python
# 最多重試 3 次
for attempt in range(3):
    if hold_coord(lat, lng): break
    time.sleep(0.5)

# hold_coord 已花 1.5s，等剩餘 interval
route_stop.wait(timeout=ivl - 1.5)
```

### 步長 / 間隔計算

```python
def step_metres(spd):
    if spd <= 10: return 8     # ~5.8s @ 5km/h
    if spd <= 20: return 12    # ~2.9s @ 15km/h
    if spd <= 40: return 18    # ~2.2s @ 30km/h
    if spd <= 70: return 30    # ~2.1s @ 60km/h
    return 55                  # ~2.1s @ 120km/h

def step_interval(spd):
    return max(2.0, (step_metres(spd) / 1000 / spd) * 3600)
```

### 隨機漫步算法

```python
pull    = min(1.0, dist_to_center / radius_m) ** 2  # 二次方：邊界才拉回
free    = (heading + uniform(-60, 60)) % 360         # ±60° 擾動
heading = free*(1-pull) + to_center*pull             # 混合
```

### 關鍵 JS 函數

**地圖**
- `setSM(lat, lng, active)` — `setLatLng()` 平滑更新（不重建 marker）

**Tunnel（實機）/ Simulator**
- `startTunnel()` → POST /tunnel/start → POST /tunnel/set-rsd
- `checkSim()` → GET /status（Simulator 版）

**定位**
- `teleport()` → POST /location/set → `triggerCooldown()`
- `clearLoc()` → POST /location/clear → `stopCooldown()`

**插值（前後端必須同步）**
- `stepM(spd)` ↔ 後端 `step_metres(spd)` — **兩份必須完全一致**
- `buildPath(pts, spd)` → `{dense, ivl, sm}`

**路線**
- `startRoute()` → POST /route/play
- `pauseRoute()` → toggle /route/pause 或 /route/resume
- `stopRoute()` → POST /route/stop（不 clear 位置）
- `startAnim(dense, ivl, loops)` — 前端進度條（`isPaused` 時凍結）
- `startRoutePoll()` — 每 1.5s 輪詢 /route/status → `setSM()` 更新 marker

**道路路線規劃**
- `setRoutingMode(m)` — m ∈ {manual, walk, cycle}
- `addRoadPoint(lat, lng)` — 呼叫 ORS API → 把道路座標加入 rPts
- `fetchRoadRoute(p1, p2, profile)` → ORS `/v2/directions/{profile}`
- ORS profile: `foot-walking`（步行）、`cycling-regular`（騎車）

**批次座標匯入**
- `parseBatchCoords(raw)` — 解析多行座標（逗號/空格/Tab 分隔）
- `batchImport()` — 解析後追加到 rPts，地圖飛到第一點

**冷卻系統**
- `calcCooldown(from, to)` — 距離 / 1.389m/s × 1.5，最長 1800s
- `startCooldown(secs)` — 啟動倒數，禁用傳送按鈕，存 localStorage
- `triggerCooldown(toPos)` — 計算冷卻，更新 cdLastPos
- `stopCooldown()` — 清除 localStorage，恢復按鈕
- localStorage keys:
  - `gps_cd_mode` — 模式設定
  - `gps_cd_state` — `{endTime, totalSecs}` 冷卻狀態
  - `gps_cd_lastpos` — 上次傳送座標（冷卻基準）
- 頁面載入時：真實位置 → `cdLastPos`（若無 localStorage 記錄）

**鍵盤控制**
- `toggleKb()` — 啟用/停用
- `kbCalcNext(dir)` — Haversine 反推計算 `kbStep`（公尺）的下一座標
- `kbHoldStart(dir)` — 300ms 後每 150ms 重複
- 支援 WASD + 方向鍵，input 框內不觸發

**儲存位置**
- localStorage key: `gps_saved_v1`
- `saveCurrentLoc()` / `loadSavedI(i)` / `delSaved(i)`

## 已知限制

| 項目 | 說明 |
|------|------|
| Pool 資源 | 路線結束後 pool 仍持有 2 個 process，需點「清除」才釋放 |
| Tunnel 斷線 | 斷線 = 定位失效，需手動重連 |
| WiFi Tunnel | 目前只支援 USB |
| Simulator | 無法安裝 App Store app，只能測試開發中的 App |
| geolocation | 必須用 localhost（`http://localhost:7788/ui`），file:// 不支援 |

## 修改注意事項

- **後端改動** → `Ctrl+C` 重啟，不支援 hot reload
- **前端改動** → F5 重整
- **調整步長** → `stepM()` 和 `step_metres()` 前後端都要改
- **不要在路線中 kill `_proc_pool`** — 核心原則
- **Simulator 座標格式** → `f"{lat},{lng}"`（逗號合併，不能分開傳）
